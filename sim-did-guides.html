Technical FAQ

Answers to common technical questions about SIM and DID configuration.

We use standard SIP headers for DID identification:

From: "Caller Name" <sip:+12125551212@voiro.online>
To: <sip:+14155551212@your-pbx.example.com>
P-Asserted-Identity: <sip:+12125551212@voiro.online>
Diversion: <sip:+12125551212@voiro.online>;reason=unconditional

The called DID number is typically in the Request-URI and To header, while the calling number is in the From and P-Asserted-Identity headers.

Failover routing can be configured through our web portal or API. You can set primary and secondary destinations for each DID:

  1. Primary destination: SIP URI or phone number
  2. Secondary destination: Backup SIP URI or phone number
  3. Failover trigger: Timeout (default 20s) or specific SIP response codes
  4. Fallback after failover: Return to primary after specified time

You can also configure geographic failover based on the caller's location.

Our SIM gateway supports the following codecs in order of preference:

G.711 ulaw
64 kbps, PSTN quality
G.711 alaw
64 kbps, International
G.729
8 kbps, Low bandwidth
GSM
13 kbps, Mobile standard

Codec negotiation is automatic, but you can force specific codecs through gateway configuration if needed.

We provide multiple monitoring options:

  • Web Dashboard: Real-time usage statistics, signal strength, and call quality metrics
  • API Access: Programmatic access to usage data and performance metrics
  • SNMP Monitoring: For integration with existing network monitoring systems
  • Email Alerts: Notifications for low balance, poor signal, or unusual usage patterns
  • Call Detail Records (CDR): Detailed logs available for download in CSV format
